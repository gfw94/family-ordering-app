# 家庭点菜网页版

这是一个网页版本。默认可以本地单机使用，也支持两种共享方式：

- 静态前端 + Firebase Realtime Database
- Python + SQLite 后端版

## 适合你的使用方式

你现在只有 PC 微信，没有微信开发者工具。

这个版本更适合你当前的环境：

- 直接用浏览器打开 index.html 就能用
- 也可以把整个文件夹发到另一台电脑继续使用
- 数据保存在浏览器本地 localStorage

## 当前能力

- 老婆 / 管理员双角色登录
- 内置一批家常炒菜，刷新后会自动补齐到现有菜单
- 老婆按日期和时段点菜
- 老婆查看自己的订单
- 管理员维护菜品
- 管理员可批量导入公开菜谱
- 管理员维护时段
- 管理员确认、取消订单
- 管理员确认后自动按日期和午餐/晚餐分别汇总需要采购的配菜
- 管理员一键复制采购清单文本
- 管理员可勾选采购项已买/未买，状态会保存在当前浏览器
- 管理员可打开打印版购物清单，直接打印或另存为 PDF
- 管理员可查看本周菜单汇总，并打印未来 7 天的菜单和采购安排
- 管理员上传本地图片作为菜品封面
- 老婆可以标记喜欢的菜，系统会推荐相似菜品

## 使用方法

1. 打开 [index.html](index.html)
2. 选择身份登录
3. 老婆进入后可以直接选时间点菜
4. 管理员进入后可以批量导入菜谱、审核订单，并自动查看采购清单

## 菜谱导入与推荐

- 当前项目已经内置一批家常炒菜，例如青椒肉丝、宫保鸡丁、番茄炒蛋、回锅肉、鱼香茄子
- 当前项目已经额外内置一批川菜家常菜，例如麻婆豆腐、水煮肉片、鱼香肉丝、辣子鸡、酸辣土豆丝、小炒黄牛肉
- 当前不直接对接下厨房，因为这类平台通常没有稳定公开接口，后续也容易失效
- 当前网页版优先尝试公开菜谱源导入
- 如果公开接口失败，会自动回退到内置精选菜谱，不会影响使用
- 管理员也可以直接复制网页里的菜谱文字，粘贴到后台做批量导入
- 老婆在点菜页点击“喜欢这道菜”后，系统会根据标签和菜系来源推荐相似菜品

推荐粘贴格式：

菜名|描述|食材1:数量,食材2:数量|标签1,标签2|午餐,晚餐

或者：

菜名
描述
食材：青椒:2个, 里脊:250g
标签：家常,炒菜
时段：午餐,晚餐

多个菜谱之间空一行。

## 采购清单规则

- 只有已确认订单才会进入采购清单
- 采购清单按日期和时段分组显示
- 同一日期同一时段内会自动累计同一种配菜的数量
- 每个采购项都可以标记为已买或未买
- 可以一键复制采购清单，直接发到微信
- 可以打开打印版购物清单，适合打印或导出 PDF

## 本周菜单汇总

- 统计未来 7 天内所有已确认订单
- 按日期和时段展示每顿饭要做的菜
- 同时带出对应采购项，方便一次性安排一周买菜
- 可以打开打印版，直接打印或导出为 PDF

## 默认身份

- 老婆：点菜用户
- 管理员：审核和采购

## 数据说明

- 菜品、时段、订单都保存在浏览器本地
- 刷新页面不会丢数据
- 换浏览器、换电脑，数据不会自动同步

## 共享版最小配置

如果你想把网址发给老婆，并且你们看到同一份订单，请使用 Firebase Realtime Database。

如果你们一人在公司局域网、一人在家里网络，推荐先把前端部署到公网，再把共享数据接到同一个 Firebase 房间，这样两边打开同一个链接就能一起用。

最小步骤：

1. 去 Firebase 创建一个项目
2. 开启 Realtime Database
3. 先用测试模式，或者给数据库规则放开读写用于家庭内使用
4. 记下数据库地址，例如：https://your-project-default-rtdb.asia-southeast1.firebasedatabase.app
5. 把当前网页部署成静态站点，例如 Netlify、Vercel 或 GitHub Pages
6. 用管理员登录，进入管理后台
7. 在“共享数据配置”里填入数据库地址和房间名，比如 family-menu
8. 勾选“启用共享模式”并点击“保存共享配置”
9. 点击“复制分享链接”发给你老婆

说明：

- 你和老婆只要打开同一个分享链接，就会连接到同一个共享数据空间
- 只要前端和共享数据源都能从公网访问，公司和家里就能同时使用
- 老婆下单后，你这边会自动同步到最新订单
- 如果你暂时不配 Firebase，系统仍然可以继续本地单机使用

## 公网共享版最小上线步骤

如果你想最快让公司和家里都能一起用，按这个顺序做就行：

1. 先把前端部署到公网静态站点，例如 Netlify、Vercel 或 GitHub Pages
2. 在 Firebase 创建一个 Realtime Database 项目
3. 先临时放开读写规则，只给家庭内使用
4. 在网页管理后台里填写 Firebase 地址和房间名，例如 family-menu
5. 勾选“启用共享模式”并保存配置
6. 点击“复制分享链接”，把同一个链接发给你老婆
7. 你在公司打开，老婆在家里打开，只要房间名一致，就会看到同一份数据
8. 等确认能正常同步后，再把 Firebase 规则收紧，避免长期开放读写

## SQLite 后端版

这个版本现在也支持 SQLite 后端。

已经准备好的文件：

- backend/server.py: Python 后端服务
- backend/schema.sql: SQLite 初始化表结构
- data/main.db: SQLite 数据库文件
- run_sqlite_backend.ps1: 一键启动脚本

本地启动方法：

1. 运行 [run_sqlite_backend.ps1](run_sqlite_backend.ps1)
2. 浏览器打开 http://127.0.0.1:8000
3. 用管理员登录后，进入管理后台
4. 在“共享数据配置”里填写房间名，例如 family-menu
5. 勾选“启用共享模式”，点击“保存共享配置”
6. 再点“复制分享链接”

说明：

- 只要前端是通过这个 Python 服务打开，页面会自动检测 SQLite 后端
- 后端把整份共享状态写进 [data/main.db](data/main.db)
- 不同房间名会存成不同的共享空间
- 如果你后面要给老婆远程访问，需要把这个 Python 服务部署到一台她能访问到的机器上，或者通过内网穿透 / VPN 暴露出来

## 如果你后面想继续升级

下一步最合适的方向是：

1. 先把网页版部署成公网共享版，支持你和老婆在公司和家里都打开同一个链接
2. 加后端和数据库，支持多人共享同一份数据
3. 再考虑封装成微信小程序
